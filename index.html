Infraestructura y Servicios

Plataformas utilizadas, límites y costos

El sistema utiliza tres servicios externos para funcionar: uno para alojar el servidor, uno para la base de datos y uno para el almacenamiento de archivos.

🚂 Railway — Servidor / Hosting

URL del sistema: https://lemescarlos-alberto-production.up.railway.app

Railway es la plataforma donde corre el servidor Node.js. Cada vez que se hace un push a GitHub, el sistema se actualiza automáticamente.

Plan actualIncluyeCosto
Hobby ✅ ACTIVO1 servicio activo, deploys automáticos desde GitHub, disponibilidad continuaUSD $5/mes
✅ El doctor ya abona USD $5/mes en Railway. El sistema corre sin interrupciones. El pago se realiza con tarjeta directamente en railway.app.

🗄️ Supabase — Base de Datos

Supabase aloja la base de datos PostgreSQL donde se guardan todos los datos: pacientes, turnos, historias clínicas, usuarios, etc.

🖼️ Cloudinary — Almacenamiento de Archivos e Imágenes

Cloudinary almacena los archivos que se adjuntan en las Historias Clínicas: resultados de estudios, imágenes, documentos PDF, etc. Los archivos se suben a la nube y se acceden desde el sistema con un enlace seguro.

Plan actualIncluyeLímite
Free25 créditos mensuales, 25 GB de almacenamiento25 GB de archivos almacenados y 25 GB de transferencia mensual
Plus225 créditos, 225 GBUSD $89/mes
✅ El plan gratuito de Cloudinary es suficiente para el consultorio durante mucho tiempo. 25 GB equivalen a miles de archivos de estudios médicos y documentos PDF.
